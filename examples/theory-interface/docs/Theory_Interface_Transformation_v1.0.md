# 摘要

理論長期以論文、書籍與講演作為主要載體。這些形式擅長保存定義、推理、證據與版本，卻未必適合讓讀者直接觀察一個理論如何運作、如何受參數影響、如何在不同條件下失敗，以及其公開說明與正式結構之間存在何種差異。

當建立公開介面、互動展示與文件門面的成本快速下降後，理論的外部形態正在發生改變。理論不再只能被閱讀，也可以被操作、比較、回放、分層查看與局部驗證。

本文提出「理論介面化」概念：

> 理論介面化，是將同一套知識分別投影為公共敘事、形式結構、可操作展示、證據狀態與版本歷史，使讀者不只接收結論，也能觀察理論的作用方式、適用邊界與尚未完成之處。

理論介面化並不取代論文，也不等於將理論簡化為視覺效果。它是一種新的知識表面：介於論述、示範、工具、出版與實驗之間。

本文進一步主張，當介面製作門檻下降後，真正稀缺的將不再是展示能力，而是語意一致性、證據治理、版本責任與長期維護。

# 1. 理論的傳統載體

理論通常透過以下形式存在：

```text
論文
書籍
講演
圖表
公式
註釋
實驗紀錄
```

這些形式具有重要優點：

- 能保存完整論證；
- 能建立正式引用；
- 能區分定義與推論；
- 能追蹤作者與版本；
- 能承載複雜內容。

但它們通常要求讀者依序閱讀，並在自己的認知中重建理論。

對於具有多層狀態、動態關係、時間演化、參數敏感性、多重分支與反事實比較的理論，純文本常不足以呈現其全部結構。

# 2. 理論介面化的定義

令一套理論為：

\[
\mathcal{T}=(D,A,R,E,B,V)
\]

其中：

- \(D\)：定義；
- \(A\)：假設；
- \(R\)：規則與推論；
- \(E\)：證據與案例；
- \(B\)：適用邊界；
- \(V\)：版本與演化歷史。

理論介面不是另一套理論，而是投影函數：

\[
\Phi_i(\mathcal{T})
\]

不同的 \(\Phi_i\) 產生公共敘事、形式定義、互動操作、實驗證據、版本歷史與限制未知等不同知識表面。因此，同一理論可以具有多種介面，但其權威來源仍須保持一致。

# 3. 從閱讀到操作

傳統理論主要要求讀者回答：作者說了什麼、推論是否成立、證據是否充分。

介面化理論則新增：改變條件後會發生什麼、哪些因素真正影響結果、模型在哪裡失效、公開版本省略了什麼、不同版本有何差異，以及哪些部分是已完成、假說或未知。

這使理論由靜態命題集合，轉變為可探索的知識空間。

# 4. 可操作摘要

傳統摘要通常由文字構成。介面化理論的摘要可以由以下元素共同構成：

```text
一個核心命題
一張結構圖
一個可調整情境
一組輸入與輸出
一個失敗案例
一個限制說明
一條通往正式文件的路徑
```

本文將此稱為「可操作摘要」。可操作摘要並不證明理論成立，而是讓讀者快速理解理論處理什麼、變數如何互動、可能產生什麼結果，以及哪些細節已被省略。

# 5. 公共層與正式層

介面化理論至少需要兩個清楚分離的層級。

公共層負責直觀說明、視覺化、簡化案例、基本互動與入口導覽，其目的不是完整，而是可接近。

正式層負責定義、假設、推導、模型、邊界、反例、證據與版本，其目的不是容易，而是精確。

公共層與正式層之間必須明確標記：此處為簡化展示、正式定義、研究假說、尚未驗證，或只在特定條件下成立。

# 6. 理論門面與理論本體

理論門面是一種入口，不是理論本體。

```text
門面 → 讓人理解、探索與找到入口
本體 → 保存正式定義、證據與版本責任
```

當門面被誤認為本體，會出現視覺比喻被誤認為數學證明、簡化模型被誤認為完整理論、互動結果被誤認為實驗證據，以及展示效果被誤認為實際能力。因此，介面化的核心不是增加裝飾，而是增加層級透明度。

# 7. 理論網站作為知識場所

一個完整的理論場所可以同時承載核心命題、公共說明、互動探索、正式文件、實驗紀錄、版本變更、限制列表、失敗案例與研究路線。

此時，它不再只是宣傳頁，而是出版場所、教學場所、實驗入口、文件索引與公開研究介面。同一場所可以同時服務初次接觸者、領域研究者、實作者、審閱者與未來維護者。

# 8. 理論介面的認識論作用

介面不只是傳播工具，也會改變理論如何被認識。它可以顯示隱藏結構、暴露參數敏感性、清楚標出省略、呈現不確定範圍與分支，並比較不同版本中的變化。

因此，好的介面不只讓理論更漂亮，也讓理論更難隱藏自身限制。

# 9. 理論介面的證據階層

```text
L0：概念敘述
L1：視覺比喻
L2：互動示範
L3：可重現模型
L4：系統性實驗
L5：外部驗證
L6：形式證明或穩健證據
```

一個網站中的可操作展示，通常只位於 L2 或 L3。它不能因為可以運作，就自動被視為已證實。

# 10. 門檻遷移

當公開介面製作成本下降後，理論公開的主要門檻會由「能不能做出網站或展示」轉移為：網站是否忠於正式理論、展示是否誤導、文件是否同步、證據是否分級、版本是否可追蹤、失敗是否公開，以及多年後是否仍能維護。

> 製作門檻下降，不等於知識責任下降。

# 11. 理論產品化與理論介面化的差異

理論產品化關注使用、效率、市場、持續服務與使用者價值；理論介面化關注理解、透明、探索、證據、版本與邊界。

兩者可以相連，但不必相同。一個理論網站可以只有門面、文件、互動摘要與公開狀態，而尚未成為完整產品。

# 12. 沉睡型理論網站

並非每套理論都需要立即發展成大型應用。可先建立「沉睡型理論網站」：具有正式入口、文件、最小展示與版本狀態，但不持續提供複雜服務。

其價值在於保存公開位置、建立可理解入口、讓未來合作快速開始、讓理論不只存在於檔案中，並等待技術、資金或應用時機成熟。

# 13. 共同框架與多理論擴展

當理論數量增加時，不應每次重新決定整體結構。可建立共同的理論介面模板：

```text
Overview
Explore
Formal Model
Experiments
Documents
Status
Limitations
History
Roadmap
```

不同理論只需填入核心命題、關鍵變數、互動方式、正式結構、證據、狀態與限制。這使理論網站由單次製作，轉變成可持續的出版制度。

# 14. 主要風險

1. 精緻介面造成視覺權威錯覺。
2. 可執行模型被誤認為真實世界證明。
3. 公共層反向支配正式理論。
4. 網站、文件與實作逐漸版本漂移。
5. 大量低成本網站快速成為無人維護的介面廢墟。

# 15. 治理原則

1. 公共層與正式層分離；
2. 每項內容標示證據狀態；
3. 互動展示標示簡化條件；
4. 正式文件具有明確版本；
5. 失敗案例與限制可被找到；
6. 網站不作為唯一真源；
7. 修改保留歷史；
8. 不確定結果標示未知；
9. 視覺比喻不得冒充形式證明；
10. 長期無人維護時應標示封存狀態。

# 16. 理論介面的成熟度

- **Level 0：文本存在。** 理論只有文件。
- **Level 1：公開門面。** 具有清楚入口與摘要。
- **Level 2：分層文件。** 公共層與正式層已分離。
- **Level 3：可操作摘要。** 具有互動展示與限制說明。
- **Level 4：證據介面。** 具有實驗、版本、失敗與重現材料。
- **Level 5：持續研究場所。** 成為社群、研究、工具與知識演化的共同場所。

# 17. 結論

理論正在從單一文本，轉化為多層知識表面。這並不表示論文將消失，而是論文將成為更大結構中的正式核心。圍繞它的公共說明、互動展示、實驗狀態、限制與版本，則形成新的理論介面。

最重要的變化不是理論變得更容易，而是更容易被看見、操作與比較，也更容易被誤解。因此，理論介面化的真正要求，不是更華麗的展示，而是更清楚的知識分層。

> **在介面成本快速下降的時代，理論的公開門檻正在降低；真正的稀缺資源，將從展示能力轉移為語意一致性、證據責任、版本治理與長期可信度。**
